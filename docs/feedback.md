## Feedback Guidelines

### What to Include

- **Clear description** of the issue or suggestion
- **Steps to reproduce** for bugs
- **Expected behavior** vs actual behavior
- **Environment details** (OS, Node version, Wrikka version)
- **Screenshots** or code examples when helpful

### Before Submitting

- Check if the issue already exists
- Search existing documentation
- Try the latest version

### What Happens Next

- We'll review your feedback within 2-3 business days
- Bug reports get prioritized by severity
- Feature requests are discussed in our planning meetings
- We'll keep you updated on the progress
