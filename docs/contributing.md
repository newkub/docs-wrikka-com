# Contributing to Wrikka

We're excited that you're interested in contributing to Wrikka Framework! This guide will help you get started.

## Getting Started

### Prerequisites

- Node.js 18 or higher
- Git
- Basic knowledge of TypeScript/JavaScript
- Familiarity with our documentation

### Setup

```bash
# Fork the repository
git clone https://github.com/YOUR_USERNAME/wrikka.git
cd wrikka

# Install dependencies
bun install

# Start development
bun dev
```

## Development Workflow

### 1. Create an Issue

- Check if the issue already exists
- Provide clear description and reproduction steps
- Use appropriate labels

### 2. Create a Branch

```bash
git checkout -b feature/your-feature-name
# or
git checkout -b fix/issue-number
```

### 3. Make Changes

- Follow our coding standards
- Write tests for new functionality
- Update documentation

### 4. Test Your Changes

```bash
# Run tests
bun test

# Run linting
bun lint

# Type checking
bun type-check

# Build check
bun build
```

### 5. Submit Pull Request

- Fill out the PR template
- Link related issues
- Request reviews from maintainers

## Coding Standards

### Code Style

- Use TypeScript for all new code
- Follow our ESLint configuration
- Use meaningful variable and function names
- Add JSDoc comments for public APIs

### File Organization

- Group related functionality
- Use consistent naming conventions
- Keep files focused and small
- Follow our project structure

### Testing

- Write unit tests for new features
- Include integration tests when appropriate
- Aim for high test coverage
- Use descriptive test names

## Documentation

### What to Document

- Public APIs and interfaces
- Configuration options
- Usage examples
- Migration guides

### Documentation Style

- Use clear, concise language
- Include code examples
- Add screenshots when helpful
- Follow our style guide

## Review Process

### What We Look For

- Code quality and maintainability
- Test coverage and quality
- Documentation completeness
- Performance implications
- Security considerations

### Review Timeline

- Initial review within 2-3 days
- Additional reviews as needed
- Merge after approval and CI passes
